Pacman

I started developing this while going through Coding Dojo's Web Fundamentals course. There was a video walk-through giving insight on the development process using Pacman as an example. I paused the video early on and started developing the app on my own. I wanted to see how my solution compared to the instructors. When I had some good bones in place, I went back and looked at the instructors solution. My solution diverged from his pretty quickly. I think mine is better of course.

Give it try. Bring up a separate version in an incognito tab to have multiple sessions.

It is JavaScript/jQuery on the client side using socket.io to send and receive information to/from an Express server. The nginx server (serving the website brucelecaptain.com) is configured to do a proxy_pass to an Express server, listening on localhost port 8080, when a request is made to /pacman or /socket.io.

Authors MEAN App

Authors is a full MEAN stack application originally developed as an excercise while I was in Coding Dojo's online bootcamp.

Deploying the app to brucelecaptain.com was a good learning experience. As with Pacman, I have an nginx server doing a proxy pass to a node server. It worked beautifully in development, but when deployed the node routing fell apart. Figuring out the problems gave me a greater understanding of node routing of an Angular app, and how to have a single version of software "work" in different environments.

Documentation of the API can be found here. A sample author class can be found here.

If this was a "real" production API, I would want to control access to the http post, put, and delete methods. This would require https and some form of validation, either requiring a key or a login.
